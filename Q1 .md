# Why does BitNet use a single global scale (i2_scale) for ternary quantization instead of per-channel or per-block scaling, and what accuracy trade-off does this introduce?






## **1 . Introduction**

BitNet’s 1-bit (ternary) quantization represents weights using values {-1, 0,  +1}. To map floating-point weights into this representation, a scaling factor (`i2_scale`) is used.

Instead of using multiple scales (per-channel or per-block), BitNet uses a **single global scale**, computed as the maximum absolute value across the entire tensor.

---

## **2 . Explanation (Concept)**

BitNet uses a single global scale (`i2_scale`) for ternary quantization by computing the maximum absolute value across the entire weight tensor. This scale is then used uniformly for all weights.Since ternary quantization already collapses values to {-1, 0, +1}, most magnitude information is lost. Therefore, the benefit of finer-grained scaling (such as per-channel or per-block) is inherently limited.

The reason for using a global scale instead of per-channel or per-block scaling is primarily efficiency.

A single scale:

* reduces memory overhead (only one float stored),  
* simplifies computation,  
* and allows better optimization in vectorized (SIMD) kernels (based on standard SIMD and memory optimization principles).

However, this comes at the cost of reduced accuracy, because different parts of the tensor may have different value distributions. A single scale cannot represent all regions equally well, leading to higher quantization error.

## **4 . Why Not Per-Channel / Per-Block Scaling?**

Using per-channel or per-block scaling would:

* require storing multiple scale values,  
* increase memory usage,  
* and add overhead during computation (loading different scales).

From a systems perspective:

* multiple scales increase memory bandwidth requirements,  
* reduce cache efficiency,  
* and complicate SIMD/vectorized kernel design.

By using a single global scale, BitNet achieves:

* faster inference,  
* simpler kernel implementation,  
* and better cache efficiency.

---

## **5 . Accuracy Cost (Trade-off)**

The drawback of global scaling is that it cannot adapt to variations in weight distributions across different channels.

For example:

* if one channel has small values and another has large values,  
* the global scale will be dominated by the larger values.

As a result:

* smaller values may get quantized to zero,  
* fine-grained information is lost,  
* and overall quantization error increases .

From a theoretical perspective:

* global scaling increases **quantization error variance**,  
* while per-channel scaling minimizes local error by adapting to distribution differences .

This leads to lower accuracy compared to per-channel or per-block scaling, which can better preserve local variations.

---

## **6 . Summary**

* BitNet uses a **single global scale (`i2_scale`)** computed from the maximum absolute value of the tensor.  
* This design reduces memory usage and simplifies computation, enabling efficient SIMD-based execution.  
* Avoiding per-channel or per-block scaling improves cache efficiency and inference speed.  
* However, global scaling cannot capture local variations in weight distributions.  
* This results in higher quantization error and some loss of accuracy compared to more granular scaling methods.
