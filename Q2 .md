## **1 . Introduction and Concept**

The MAD quantizer is responsible for converting floating-point weights into a ternary representation {-1, 0,  +1}. Instead of storing these values directly, the implementation uses an encoded format (`q8`) with values {0, 1, 2}, where:

* 0 represents  -1  
* 1 represents 0 (middle value)  
* 2 represents  +1

This quantization enables efficient low-bit computation while reducing memory and computational overhead.

A key design decision in this quantizer is the handling of **near-zero values** (i.e., values where the absolute magnitude is less than 1e-6). These values are explicitly mapped to the **middle ternary value (0)** instead of being assigned based on their sign.

---

## **2 . Why Near-Zero Values Are Mapped to the Middle Ternary Value**

Near-zero weights effectively represent **no contribution** to the model output. In neural networks, the output is computed as a weighted sum:

Output ≈ Σ (weight × activation)

If a weight is extremely small:

Weight ≈ 0 → Contribution ≈ 0

To preserve this behavior after quantization, such values must be mapped to **0 (neutral value)**.

---

### **Key Reasons**

---

### **a. Preserving Semantic Meaning**

Near-zero values indicate negligible importance. Mapping them to 0 ensures that no artificial positive or negative signal is introduced during quantization.

---

### **b. Maintaining Distribution Symmetry**

Neural network weights are typically centered around zero (mean ≈ 0). Mapping near-zero values to 0 preserves this symmetry and prevents skewing the distribution toward positive or negative values  [2 ].

---

### **c. Reducing Quantization Noise**

Quantization introduces approximation error. Very small values often arise due to noise or numerical precision limitations. Mapping them to zero acts as implicit denoising and avoids amplifying insignificant values  [1 ].

---

### **d. Preventing Error Accumulation**

In matrix multiplication:

Output = Σ (wᵢ × xᵢ)

Even small misrepresentations can accumulate across many terms. Incorrectly assigning near-zero values can significantly distort the final output  [1 ].

---

## **4 . Code Explanation**

The condition:

fabs(src [i ])  < 1e-6

checks whether the weight value is extremely close to zero.

---

### **If the condition is true:**

q8 [i ]  = 1

This assigns the encoded value `1`, which corresponds to the **middle ternary value (0)**.  
This ensures that near-zero values contribute nothing to the output.

---

### **If the condition is false:**

* Positive values → q8 [i ]  = 2 → represents  +1  
* Negative values → q8 [i ]  = 0 → represents  -1

Thus, the encoding scheme is:

* 0 →  -1  
* 1 → 0  
* 2 →  +1

This confirms that near-zero values are intentionally mapped to the **neutral element**, while other values are mapped based on their sign.

---

## **5 . What Happens If q8 [i ]  = 0 Is Used Instead?**

If near-zero values are encoded as:

q8 [i ]  = 0

they are interpreted as:

 -1 (strong negative value)

---

## **6 . Consequences of Incorrect Encoding**

---

### **a. Sign Distortion**

A very small value such as 0.000001 would be incorrectly mapped to  -1, introducing a large negative signal.

---

### **b. Bias Introduction**

Mapping many near-zero values to  -1 shifts the overall weight distribution toward negative values, breaking symmetry  [2 ].

---

### **c. Loss of Neutrality**

Instead of representing “no contribution”, these values now introduce strong negative contributions.

---

### **d. Error Amplification**

In computations like:

Output  = Σ (wᵢ × xᵢ)

These incorrect mappings accumulate across multiple terms, leading to significant deviation in the final output  [1 ].

---

### **e. Reduced Model Accuracy**

The quantized representation no longer reflects the true distribution of weights. This increases quantization error and degrades overall model performance  [1 ].

---

## **7 . Final Insight**

Mapping near-zero values to the middle ternary value ensures that:

**“No contribution remains no contribution after quantization.”**

If these values are instead mapped to ±1, insignificant noise is converted into strong signals, leading to bias, accumulated error, and reduced model accuracy.

---

## **8 . Summary**

* Near-zero values represent negligible contribution and must map to **0 (neutral value)**.  
* BitNet explicitly assigns `q8[i] = 1` to preserve this behavior.  
* This maintains distribution symmetry and reduces quantization noise.  
* Mapping near-zero values to  -1 instead introduces bias and incorrect signals.  
* This leads to error accumulation and reduced model accuracy.
